# Support

HUMAN LEGAL REVIEW REQUIRED

This repository is being prepared for a first paid self-hosted release sold as a one-time purchase.

## Default support posture

- perpetual self-hosted license for the purchased release build
- limited updates, not an open-ended subscription
- buyer-operated runtime, not a hosted service

## Draft support boundary

The first paid release should support:

- first install on the documented local/Docker path
- `.env` setup and doctor/smoke validation
- documented upgrade flow
- documented backup/restore flow
- notification configuration sanity
- filing-pipeline troubleshooting within the documented runtime shape

The first paid release should not imply:

- managed hosting
- broker or trading advice
- SEC interpretation services
- custom feature work
- unlimited ongoing consulting

## Draft update boundary

Suggested commercial default for this first release:

- bug-fix and documentation updates for the purchased minor line for a limited window
- no promise of indefinite feature expansion

Suggested draft window for review:

- 6 months of maintenance updates from purchase date

This is a commercial policy placeholder and must be confirmed before sale.

## What a buyer should include in a support request

- install method: Docker or local `uv run`
- redacted `.env`
- `SEC_USER_AGENT`
- output of `make doctor`
- output of `make smoke`
- relevant `/settings` and `/advanced` screenshots
- the latest `/errors` entry, if present

## Support channel

Use the support email or contact method provided at the point of sale.
