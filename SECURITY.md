# Security

## Security posture

- Localhost-first in v1.
- Secrets are environment-variable based.
- Secrets are not stored in the database.
- The product is single-user and single-process in v1.
- External notification providers are opt-in.

## Reporting

If you believe you found a security issue, report it privately rather than posting sensitive details in a public issue.

## Buyer expectations

- Keep `.env` private.
- Rotate external service credentials if you suspect exposure.
- Run the app on a machine you control.

## HUMAN LEGAL REVIEW REQUIRED

Final vulnerability disclosure timing, support contact, and any security warranty language should be reviewed before sale.
