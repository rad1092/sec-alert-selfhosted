# Capacity Benchmark Plan

The first paid release keeps the documented watchlist envelope conservative.

## Current support posture

- `25` validated
- `50` hard cap
- `100` unsupported

## Future benchmark plan

If higher issuer counts are explored later, measure:

- queue depth and backlog growth
- average ingest duration
- repair/backfill duration
- filing throughput under mixed 8-K and Form 4 activity
- UI responsiveness with a populated local archive

## Benchmark scenarios

- 10 issuers
- 25 issuers
- 50 issuers
- 75 issuers as experimental only
- 100 issuers as unsupported stress only

## Reporting

Any future claim above the current documented posture should be backed by repeatable local benchmark notes and updated troubleshooting/install guidance.
